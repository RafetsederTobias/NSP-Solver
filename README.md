# NSP-Solver — An ASP-based Nurse Scheduling System

> Bachelor's thesis project at Johannes Kepler University Linz (JKU)  
> *Ein ASP-basierter Ansatz zur Lösung des Nurse Scheduling Problems mit Clingo*

A full-stack web application for automated nurse schedule generation and short-notice rescheduling, powered by Answer Set Programming (ASP) with [Clingo](https://potassco.org/clingo/).

---

## Overview

The **Nurse Scheduling Problem (NSP)** involves assigning nursing staff to workdays while respecting organisational requirements and individual preferences. This system automates that process for a hospital ward using a declarative ASP encoding and exposes the result through an interactive web interface.

**Key capabilities:**

- Generate a monthly duty schedule for all staff in a ward
- Enforce hard constraints (qualification requirements, staffing minimums, one station per person per day)
- Optimise soft constraints (fair workload distribution across stations)
- Reschedule on short notice when a staff member is unexpectedly absent
- Proactively compute alternative plans as a contingency

## Architecture

The application follows a three-tier architecture:

```
Angular SPA  ──HTTP/REST──▶  FastAPI Backend  ──SQL/PostgREST──▶  Supabase (PostgreSQL)
                                    │
                               ASP-Service
                            (Clingo solver)
```

The **ASP-Service** inside the backend translates database records into ASP facts, runs Clingo, and writes the resulting assignments back to the database. The frontend communicates exclusively via REST.

## Features

### Scheduling

A monthly plan is generated by clicking *Monat planen*. Before the solver runs, constraints can be configured per employee:

- **Vollzeit** — full-time (all working days in the month)
- **Exakte Anzahl** — exact number of working days
- **Spannweite** — minimum / maximum range
- **Fixe Tage** — days the employee must be scheduled
- **Gesperrte Tage** — days the employee is unavailable

Three scheduling modes are available:

| Mode | Behaviour |
|---|---|
| Überschreiben (default) | Overwrites the currently loaded plan |
| Neuer Plan | Creates a new plan without touching the active one |
| Alternativplan | Computes an additional contingency plan alongside the main plan |

### Rescheduling

Via *Spontaner Ausfall*, absences can be entered per employee. The solver then updates only the affected future assignments, keeping all other assignments stable. A summary dialog shows exactly which assignments were removed and which were added.

## Installation

### 1 — Clone the repository

```bash
git clone <repository-url>
cd <repository-folder>
```

### 2 — Frontend

```bash
cd frontend
npm install
ng serve
```

The frontend is available at `http://localhost:4200`.

> If Angular CLI is not yet installed:
> ```bash
> npm install -g @angular/cli@21.2.6
> ```

### 3 — Backend

```bash
cd backend
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
uvicorn main:app --reload
```

The backend API is available at `http://localhost:8000`.

### 4 — Database

Create a `.env` file in the `backend/` directory and add your Supabase connection credentials:

```env
SUPABASE_URL=...
```

## Usage

1. **Stammdaten anlegen** — create employees (*Mitarbeiter*), stations (*Stationen*), and skills (*Kompetenzen*) via the sidebar navigation.
2. **Dienstplan erstellen** — open the *Kalender* view, click *Monat planen*, configure per-employee constraints, and click *Solver starten*.
3. **Tagesdetails** — click any day in the calendar to see and manually adjust assignments for each station.
4. **Rescheduling** — click *Spontaner Ausfall*, mark the unavailable days for affected employees, and click *Neu planen*.
